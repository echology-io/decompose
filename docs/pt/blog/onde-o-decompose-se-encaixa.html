Onde o Decompose
Realmente Se Encaixa

Não é o produto completo. É a etapa estrutural que torna tudo o mais possível.

20 de fevereiro de 2026
6 min de leitura

As pessoas encontram o Decompose e acham que encontraram o produto. Não encontraram. O Decompose é uma etapa em um sistema muito maior. É o primitivo cognitivo open-source — a etapa estrutural que acontece antes de o raciocínio começar.

O produto completo é o AECai. Esta publicação explica como eles se conectam, o que cada parte faz e por que tornamos open-source a parte que tornamos.

O pipeline

Todo documento que entra no AECai passa por cinco etapas. O Decompose é a Etapa 4.

Etapa 1 Parsear PDF, DOCX, DXF, imagens → texto bruto
↓
Etapa 2 Classificar Tipo de documento, domínio, indicadores de risco
↓
Etapa 3 Enriquecer Normas, PII, cronogramas, financeiros, contratos
↓
Etapa 4 — Open Source Decompose Chunking semântico, autoridade, risco, pontuação de atenção
↓
Etapa 5 Indexar Embeddings vetoriais → Qdrant → arquivo pesquisável

As Etapas 1–3 preparam o documento. A Etapa 4 lhe dá estrutura. A Etapa 5 o torna pesquisável. O Decompose é a dobradiça — é onde o texto bruto se torna unidades semânticas classificadas e pontuadas sobre as quais tudo downstream pode raciocinar.

O que o Decompose faz (e não faz)

O Decompose é uma biblioteca autônoma. Você pode pip install decompose-mcp agora mesmo e usá-lo sem o AECai, sem a Echology, sem mais nada. Ele faz exatamente uma coisa:

Texto entra, unidades semânticas classificadas saem. Autoridade, risco, atenção, entidades. Sem LLM. Determinístico.

Só isso. Ele não parseia PDFs. Não executa OCR. Não pesquisa um banco de dados vetorial. Não conversa com um LLM. Recebe texto e retorna estrutura.

O Decompose faz

  • Dividir texto em unidades semânticas
  • Classificar autoridade (obrigatório, diretivo, permissivo)
  • Pontuar risco (segurança, financeiro, conformidade)
  • Calcular atenção (prioridade de 0–10)
  • Extrair entidades (normas, datas, valores)
  • Sinalizar conteúdo irredutível
  • Executar em ~14ms, deterministicamente

O AECai adiciona

  • Parsear mais de 15 formatos de arquivo (PDF, DOCX, DXF, imagens)
  • Classificação de documentos com IA via LLM local
  • Referência cruzada de normas por jurisdição
  • Detecção e redação de PII
  • Plugins de análise de cronograma e financeira
  • Busca vetorial em todo o seu arquivo
  • Verificação, trilhas de auditoria e certificados
  • Geração de scripts CAD/BIM
  • Orquestração de fluxos de trabalho (Temporal)
  • Servidor de API completo (FastAPI)

O Decompose é o primitivo cognitivo. O AECai é a plataforma de inteligência construída sobre ele.

Os três motores

O AECai tem três motores. Cada um lida com uma fase diferente da inteligência de documentos. O Decompose vive dentro do primeiro.

Vanta é o motor de ingestão. Ele lê documentos, os classifica, executa plugins de enriquecimento e chama o Decompose para quebrá-los em unidades semânticas. Parsear → Classificar → Enriquecer → Decompose → Indexar. Esse é o pipeline do Vanta.

Aletheia é o motor de verificação. Ele verifica unidades decompostas em relação a normas específicas por jurisdição, valida referências cruzadas, executa verificações de consistência e emite certificados de auditoria. Quando o AECai diz que um documento é aprovado, o Aletheia é a razão.

Daedalus é o motor de recuperação e ação. Ele pesquisa o arquivo indexado, encontra padrões entre documentos e gera saídas — incluindo scripts para Civil3D e Revit. Quando um engenheiro pergunta "mostre-me toda especificação de fundação que referencia ACI 318," o Daedalus responde.

Por que tornamos esta parte open-source

Três razões.

1. Ela ensina a arquitetura. Se você entende o que o Decompose faz — dividir texto em unidades classificadas com pontuações de autoridade, risco e atenção — você entende o padrão de design fundamental por trás de todo o AECai. Estrutura antes do raciocínio. Classificação antes da geração. Pré-processamento determinístico antes da inferência probabilística.

2. É útil por conta própria. Você não precisa da plataforma AECai completa para se beneficiar da decomposição. Se você está construindo um pipeline RAG, um agente de IA ou qualquer sistema que processa documentos — o Decompose melhora o seu modelo ao lhe dar estrutura antes de começar a pensar.

3. É a entrada. Equipes que começam com o Decompose aprendem o vocabulário: níveis de autoridade, categorias de risco, pontuações de atenção, irredutibilidade. Quando estiverem prontas para a plataforma completa — parseamento, verificação, busca, auditoria — elas já falam a linguagem.

O quadro completo

Aqui está a forma mais simples de pensar sobre isso:

Echology (a empresa) ├── Decompose — open source, gratuito, pip install └── AECai — plataforma comercial ├── Vanta (ler + classificar + decompor + indexar) ├── Aletheia (verificar + auditar + certificar) └── Daedalus (buscar + recuperar + gerar)

O Decompose é o primitivo cognitivo. O AECai é o que acontece quando você pega esse primitivo e constrói uma plataforma de inteligência de documentos empresariais ao redor dele. Vanta, Aletheia e Daedalus são os três motores que fazem essa plataforma funcionar.

Tudo roda localmente. Sem nuvem. Nenhum dado sai do seu prédio. Isso não é uma limitação — para os setores que atendemos, é o requisito.

Onde se encaixa no AEC

Empresas de AEC lidam com documentos que têm peso legal. Especificações, contratos, submittals, RFIs — não são artigos, são obrigações. O Decompose transforma cada um deles em unidades pontuadas e classificadas sobre as quais sistemas downstream podem raciocinar.

Especificações

Revisão de Especificações

Uma especificação estrutural de 200 páginas chega. O Decompose encontra as 40 cláusulas obrigatórias, sinaliza os 12 requisitos críticos de segurança, extrai cada referência ASTM e ACI, e pontua cada seção por atenção. Seu engenheiro lê 30 páginas em vez de 200.

Contratos

Análise de Contratos

Multas por atraso, retenção, indenização, seguros mínimos — enterrados no texto padrão. O Decompose traz à superfície cada obrigação financeira e sinaliza cláusulas irredutíveis que não podem ser parafraseadas. Seu gerente de projeto vê o dinheiro no primeiro dia.

Submittals

Processamento de Submittals

Fichas técnicas de produtos, desenhos de fabricação, certificados de materiais. O Decompose classifica quais seções carregam requisitos de conformidade obrigatórios versus conteúdo informativo de marketing. Encaminhe as seções de conformidade ao seu engenheiro. Pule o resto.

RFIs

Triagem de RFIs

Um RFI referencia três seções de especificação e pergunta sobre uma substituição. O Decompose quebra as seções referenciadas em unidades, as pontua e traz à superfície as restrições obrigatórias que a substituição precisa satisfazer. Sua resposta é fundamentada, não especulativa.

QA/QC

Conformidade com Normas

Alimente suas normas internas no Decompose. Alimente os documentos recebidos no Decompose. Compare as unidades estruturadas. Cada lacuna entre "o que exigimos" e "o que eles enviaram" torna-se visível e auditável.

Treinamento

Fine-Tuning de Modelos

Cada unidade decomposta é uma amostra de treinamento rotulada: autoridade, risco, atenção, irredutibilidade. Construa um corpus de documentos AEC, decomponha todos eles e você terá dados de treinamento estruturados para IA específica do domínio. Sem rotulagem manual.

Onde se encaixa além do AEC

O Decompose foi construído para AEC, mas a arquitetura é universal. Qualquer setor que processa documentos com obrigações, risco e requisitos de conformidade pode usá-lo. Aqui estão os fluxos de trabalho que estamos vendo.

Jurídico

Escritórios de advocacia processam contratos, regulamentos, peças processuais e documentação de conformidade. O Decompose classifica níveis de autoridade ("deverá" vs. "deveria" vs. "pode"), extrai cada obrigação e prazo, e sinaliza cláusulas que devem ser preservadas literalmente. Fluxo de trabalho: Documento de entrada → Decompose → encaminhar cláusulas obrigatórias para revisão do advogado, pular seções informativas → gerar resumo de obrigações com trilha de auditoria.

Seguros

Subscritores leem apólices, endossos e documentos de sinistros o dia todo. O Decompose traz à superfície cada limite de cobertura, exclusão e cláusula condicional com pontuações de atenção. Fluxo de trabalho: Documento de apólice → Decompose → extrair termos e condições financeiras → sinalizar exclusões e limitações → comparar com envios de sinistros para lacunas de cobertura.

Saúde & Ciências da Vida

Protocolos clínicos, submissões regulatórias e documentação de conformidade carregam requisitos críticos de segurança. O Decompose identifica procedimentos obrigatórios, sinaliza valores de dosagem e limiar como irredutíveis, e pontua o risco por seção. Fluxo de trabalho: Documento de protocolo → Decompose → sinalizar seções críticas de segurança → preservar dosagens literalmente → encaminhar para revisão de conformidade.

Governo & Defesa

Aquisições federais, SOWs e orientações regulatórias seguem linguagem de autoridade estrita (FAR, DFARS, NIST). O Decompose classifica cada requisito por nível de autoridade e extrai referências a normas. Fluxo de trabalho: RFP ou SOW → Decompose → extrair cada requisito "deverá" → mapear para lista de verificação de conformidade → identificar lacunas na resposta da proposta.

Serviços Financeiros

Documentos de empréstimo, registros regulatórios e relatórios de auditoria contêm obrigações enterradas em prosa densa. O Decompose extrai cada valor monetário, percentual e prazo, sinaliza seções de risco financeiro e pontua a atenção. Fluxo de trabalho: Registro regulatório → Decompose → extrair entidades financeiras → sinalizar obrigações de conformidade → alimentar dados estruturados para modelos de risco.

Qualquer Pipeline RAG

Se você está construindo geração aumentada por recuperação (RAG) para qualquer domínio — o Decompose é a etapa de pré-processamento que faz funcionar melhor. Em vez de incorporar chunks de texto bruto, incorpore unidades semânticas classificadas. Sua busca vetorial retorna cláusulas obrigatórias e requisitos de segurança em vez de parágrafos aleatórios.

# Generic workflow for any industry # 1. Decompose the document from decompose import decompose_text units = decompose_text(document_text)["units"] # 2. Route by priority critical = [u for u in units if u["attention"] >= 3.0] review = [u for u in units if u["risk"] != "informational"] skip = [u for u in units if u["attention"] < 0.5] # 3. Protect what can't be changed verbatim = [u for u in units if u["irreducible"]] # 4. Send only what matters to your LLM for unit in critical: response = llm.analyze(unit["text"], unit["metadata"])

O padrão é sempre o mesmo: decompor primeiro, rotear por estrutura, proteger o que é irredutível, e enviar apenas o conteúdo de alta atenção ao modelo. O setor muda. A arquitetura não.
