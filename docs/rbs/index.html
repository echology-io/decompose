Pipeline Overview

Policy QC automates the manual document comparison process. Each document in a policy set is parsed, fields are extracted, and values are cross-checked against every other source. Discrepancies are flagged with exact source attribution.

1 Document Classification

Before parsing, every uploaded file is classified by filename pattern. QC checklists, SOPs, document maps, and baseline outputs are automatically filtered out — only actual policy source documents (Quote, Policy, Application, TAM export) proceed to extraction. Each source gets a standardized label for the comparison grid.

Source Documents

Approved Bound Quote, Policy, Application, TAM (Agency System), Binder, ACORD Forms — classified and labeled automatically from filename patterns.

Filtered Out

QC Checklists (XLSX), Policy Quality Control SOPs (DOCX), Baseline Outputs, Document Maps — excluded from comparison to prevent noise.

2 Document Parsing

Each source file is routed to the appropriate parser based on file type. PDFs with embedded text are extracted natively using pdfplumber. Scanned PDFs (images) are converted to 300 DPI images and processed through Tesseract OCR. DOCX files with embedded screenshots (like TAM exports) have their images OCR'd individually.

Native PDF

Text-based PDFs are extracted with pdfplumber — fast, high accuracy, preserves layout structure. Used for quotes, binders, and machine-generated policies.

Scanned PDF (OCR)

pdf2image converts pages to 300 DPI images. Tesseract 5.5 runs OCR per page. Handles scanned applications, endorsements, and form schedules.

Office Documents

python-docx extracts paragraphs and tables from DOCX. openpyxl reads XLSX cell values. Used for TAM exports, QC checklists, and document maps.

3 Field Extraction

Once raw text is extracted, regex-based patterns identify and extract insurance-specific fields. Patterns are tuned per document type — ACORD forms, RISCOM quotes/binders, carrier policies, and agency management exports all have different label conventions.

Post-extraction cleaning strips trailing noise ("Producer:", "Commission: 10.00%"), removes form label artifacts, and validates extracted values (e.g., policy numbers must contain digits, premiums must exceed $500).

4 Normalization

Before comparison, all values pass through a normalization layer that accounts for formatting differences that are not true discrepancies. This implements the rules from the existing Policy QC SOP.

  • ✓ Date formats unified: 6/5/2024 → 06/05/2024, separators normalized (to / through / – → -)
  • ✓ Entity name variants: , LLC = LLC = L.L.C.
  • ✓ Address abbreviations: Suite = Ste, Boulevard = Blvd
  • ✓ Punctuation stripped: periods, commas, extra spaces
  • ✓ Case-insensitive comparison across all fields
  • ✓ SOP exceptions: middle initials, street suffixes, DBA order, vehicle model variants

5 Cross-Source Comparison

Every extracted field is compared across all N sources simultaneously. Each field receives one of four statuses:

Match

All sources agree on the normalized value. No action needed.

Partial

Values are consistent where present, but one or more sources don't list the field. Flagged for awareness — may be expected (not all carriers include every field).

Discrepancy

Two or more sources disagree on the normalized value. Every conflicting pair is identified with exact source attribution.

Missing

Field not found in any source. May indicate extraction gap or genuinely absent data.

7 Technology Stack

Runtime

Python 3.11 — server and all processing
FastAPI + uvicorn — async HTTP server

Document Parsing

pdfplumber — native PDF text
pdf2image + poppler — PDF → image
Tesseract 5.5 — OCR engine
python-docx / openpyxl — Office files

Extraction Engine

Deterministic regex patterns — no LLM, no API calls, no model inference. Same input always produces same output. Fully auditable.

Deployment

Runs locally — no data leaves the network. Single Python process, no database, no external dependencies. Can be containerized with Docker.

Key Properties

Deterministic

No LLM. No randomness. Same document always produces the same extracted fields and comparison results. Critical for audit and compliance.

Air-Gapped

Zero external API calls. No data sent to cloud services. All processing happens on-premise. Meets E&O and client confidentiality requirements.

Extensible

New carrier formats added as regex pattern sets. New coverage lines added as field templates. New document types plug into the parser layer.

Fast

Pre-extracted comparisons return in milliseconds. OCR processing runs at ~2-5 seconds per page depending on scan quality and resolution.
