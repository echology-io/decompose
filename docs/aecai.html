The Pipeline

Five stages. Any document.

Every document flows through the same pipeline. PDF, DOCX, CAD, images — 15+ formats supported.

Stage 1

Parse

TextExtractor reads 15+ formats — PDF, DOCX, DXF, Markdown, CSV, HTML, RTF, images via OCR. Outputs raw text, cleaned text, page splits, metadata.

Stage 2

Classify

AI-first classification via local LLM. Detects document type, domains, entities, risk indicators, authority level. Regex fallback when AI is unavailable.

Stage 3

Enrich

Five built-in plugins: standards crossref, PII detection, timeline extraction, financial analysis, contract analysis. Priority-ordered and extensible.

Stage 4

Decompose

Header-aware semantic chunking. Each unit gets its own classification — authority level, content type, risk relevance. Documents become structured data.

Stage 5

Index

768-dim vector embeddings stored in Qdrant. Search your entire archive by meaning, not keywords. Filterable by jurisdiction, discipline, risk level.

Three Engines

Engine

Vanta

Reads, classifies, and decomposes documents. Simulation-aware pipeline orchestration: deterministic message passing, nested document contexts, self-improving detection patterns, attention-budget scheduling.

Engine

Aletheia

Verifies output quality and checks standards against jurisdiction adoptions. Causal consistency graphs, multi-channel error correction, Merkle-tree tamper detection, counterfactual decision logging.

Engine

Daedalus

Retrieves patterns from processed archives and generates CAD/BIM scripts. Civil3D and Revit script generation. Holographic redundancy encoding, topological data segregation.
